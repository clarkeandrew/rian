Rian

A tiny, Flyway-compatible database migration runner.

Rian applies SQL migrations and records them in a Flyway-compatible flyway_schema_history table. It ships as a single static binary — no JVM, no runtime — so it drops into a minimal container or a CI job. Point it at a database an existing Flyway already manages and it reads the same migration files and computes the same checksums.

PostgreSQL and MySQL are supported.

Install

Download a prebuilt binary for your platform:

Latest release →

Builds are published for Linux, macOS, and Windows on both amd64 and arm64.

Or pull the container image:

docker pull ghcr.io/clarkeandrew/rian:latest

Or install from source with Go:

go install github.com/clarkeandrew/rian/cmd/rian@latest

Usage

rian migrate \
  -url jdbc:postgresql://localhost:5432/app \
  -user app -password secret \
  -locations filesystem:./sql

rian info       # show applied vs. pending migrations
rian validate   # check applied checksums against the files
rian repair     # clear failed entries from the history table

Configuration is read from flyway.conf files, FLYWAY_* environment variables, and command-line flags, in that order of precedence (flags win). Both Flyway's single-dash -url and the GNU-style --url are accepted, so existing Flyway commands work unchanged.

Flyway compatibility

Rian is a drop-in for the open-source Flyway engine: the same schema-history format, the same V<version>__name.sql and R__name.sql files, and the same CRC32 checksums. Flyway can migrate a database and Rian can continue it, or the reverse.

One caveat carried over from the database itself: MySQL commits DDL implicitly, so a migration that fails partway cannot be rolled back. Like Flyway, Rian marks it failed and requires rian repair before you retry.

The name

Rian is an Irish name, usually read as “little king” — from the Irish rí, “king.” It's short, lowercase-friendly, and easy to type a hundred times a day.
